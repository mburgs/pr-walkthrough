# Fixtures

Hand-crafted sample data so every stream can develop end-to-end without standing up its peers. Each fixture validates against the Pydantic models in `contracts/schemas.py`.

## `pr_small/` — 3-file PR

A made-up "rotate session tokens on privilege change" PR. Roughly 50 LOC of change. Small enough that streams iterate fast on it.

```
pr_small/
  metadata.json              # PRMetadata
  diff.json                  # list[Hunk]
  tour_plan.json             # TourPlan (3 chunks)
  chunks/
    c1.narration.json        # ChunkNarration
    c2.narration.json        # ChunkNarration
    c3.narration.json        # ChunkNarration
  follow_up_example.json     # {follow_up: FollowUp, answer: FollowUpAnswer}
  flags_example.json         # list[Flag]
```

## What's intentionally missing

- **Audio fixtures.** No `.wav` files. The frontend should use a silent or any-sample WAV during development; once stream 4 (TTS) lands, real audio replaces the placeholder.
- **`pr_medium/`.** Will be generated by stream 3 once Claude is wired up, against a real PR. Hand-crafting it isn't worth the calories.

## Validation

```bash
python -m contracts.validate_fixtures
```

This loads every JSON file under `fixtures/` and parses it against the matching schema. CI runs it.
